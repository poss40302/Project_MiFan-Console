🚀 Project MiFan-Console 開發藍圖 (Windows Local 輕量化版)
📌 專案目標
打造一款具備科技美學、輕量化、且能動態展開控制面板的 Windows 桌面懸浮小工具，用於遠端控制小米電風扇。


🛠️ 技術棧選型
核心語言: Python 3.x
通訊協議: python-miio (負責與設備進行加密 UDP 通訊)
GUI 框架: PyQt6 / PySide6 (支援半透明、自定義繪圖與動畫)
非同步機制: asyncio (確保網路請求不阻塞 UI 介面)


⚙️ 核心功能模組
1. 智能偵測與動態 UI 邏輯
狀態輪詢: 程式啟動後需自動偵測風扇狀態。
動態展開機制:
關閉狀態: 僅顯示一個具備「青藍色霓虹發光」效果的電源符號按鈕與停止的風扇扇葉圖示。
啟動狀態: 點擊電源鍵啟動後，電源符號按鈕顯示啟動、風扇扇葉圖式開始轉動，UI 自動展開（加載動畫）顯示進階控制項。
2. 進階控制項功能
模式切換: 實作 Slider 式的布林開關，切換「直吹風」與「自然風」。
智能檔位顯示:
直吹風模式: 顯示 1~100% 的滑動條，滑動條末端顯示當前數值。
自然風模式: 顯示檔位切換按鈕（例如 1-4 檔）。
擺動控制: 實作 Slider 式的布林開關，控制風扇首部擺動。
3. 視覺與交互設計 {視窗介面}
科技風格視覺:
採用圓形或膠囊型半透明設計。
加入青藍色系 (Cyan/Neon Blue) 的炫光或霓虹線條邊框。
視窗層級: 設為 Normal 略高於桌面，但不強制永遠置頂。
交互與記憶:
左鍵單擊: 電源啟動/關閉切換。
滑鼠滾輪: 向上滾動增強風量，向下減弱(僅為啟動狀態實有效)。
長按拖曳: 自由改變位置。
座標記憶機制: 程式結束時自動儲存座標至 config.json。
多螢幕防丟失: 啟動時偵測當前座標是否在有效螢幕範圍內，若擴展螢幕失聯，自動重置於主螢幕中央。


🗒️ Agent 協作指令 (Prompts)
第一階段：後端通訊與狀態回傳
「請使用 python-miio 編寫一個非同步控制器。需包含自動偵測型號的功能，並實作一個持續輪詢 (Polling) 狀態的機制，以便 UI 判斷何時該展示電源按鈕或展開控制面板。」
第二階段：自定義科技感 UI
「請使用 PyQt6 繪製一個無邊框的懸浮窗。請用繪圖引擎 (QPainter) 實現青藍色霓虹線條的發光效果。實作一個動畫效果：當 fan_on 變數為真時，視窗從圓形鈕向外展開成膠囊型面板，並顯示 Slider 開關。」
第三階段：座標記憶與螢幕偵測
「請實作視窗位置的讀取與寫入功能。在 showEvent 中檢查座標是否超出當前所有可用螢幕的範圍（例如原本在已拔除的螢幕上），如果是，請將視窗強制移動至主螢幕座標 (100, 100)。」


📦 打包指令 (Deployment)
開發完成後，請在 Antigravity 終端執行以下指令以生成獨立 EXE：

# 安裝打包工具 (基於 CleanSight 專案建議，嚴防卡巴斯基誤殺)

pip install nuitka

# 執行打包 (隱藏控制台窗口，單一檔案模式，啟用 PyQt6 支援)

nuitka --onefile --windows-disable-console --enable-plugin=pyqt6 --output-dir=build --windows-icon-from-ico=app_icon.ico main.py
